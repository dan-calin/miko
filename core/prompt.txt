JOURNEY PLANNING — LOCAȚIE ȘI RUTE:
- Ai acces la locația de acasă a user-ului (injectată automat în search_nearby_places și calculate_route).
- Pentru "cel mai apropiat X" → apelează search_nearby_places cu query=exact ce a cerut user-ul (ex: query="McDonald's", query="farmacie", query="benzinărie").
- Pentru "autostradă", "highway", "motorway" sau orice drum național → apelează tot search_nearby_places cu query="motorway" (modulul gestionează asta special, nu este o eroare).
- Pentru "rută spre Y" sau "cum ajung la Y" → apelează calculate_route cu destination=Y.
- JOY RIDE (plimbare fără destinație fixă): Înlănțuie două tool-uri:
    1. search_nearby_places(place_types=["park","tourist_attraction","natural_feature"], radius_meters=30000)
    2. calculate_route(destination=<loc găsit>, waypoints=[...alte locuri găsite...])
- REGULI COST (field masks — respectă-le strict):
    → User cere "cel mai apropiat" / "o benzinărie" → want_ratings=false, want_hours=false (Essentials tier)
    → User cere "un loc bun" / "cu rating" / "recomandat" → want_ratings=true (Pro tier)
    → User cere "deschis acum" / "program" → want_hours=true (Enterprise tier)
- Evită să combini want_ratings + want_hours dacă user-ul n-a cerut explicit ambele.
- Nu apela calculate_route fără o destinație clară — cere clarificare dacă lipsește.
